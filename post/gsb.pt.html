GSB

Context

The pharmaceutical laboratory GSB (Galaxy Swiss Bourdin) wishes to make available to the medical visitors (its sales representatives) a Web application allowing to centralize the reports of visit to the practitioners (doctors). This information base will be used for the elaboration of the communication approach with the practitioners and will give an individual and synthetic vision of the representation activity. To assist with reporting, the application will provide a description of the laboratory’s products, specific contact information for practitioners and detailed information about them.

Specifications for the GSB project.

  • Form of the object: Initially in the form of a secure and authenticated website, the application may become in the future a tool accessible from mobile terminals (tablets or phones).

  • Accessibility/Security: The home page must propose the authentication part of the visitor. Once authenticated, the visitor will be able to access the various features of the application. His name and first name will be displayed in order to visualize his identity. An option will allow him to log out.

  • Ergonomics: The website will be like the public website of the company (see ergonomics of the html pages already made) in terms of ergonomics and colors, but will have a business and productivity vocation. We will therefore limit all superfluous graphic elements (no animated menu, no flash module). We can improve or control the user interface with Javascript code.

  • Environment: The web application will be developed in a server-side scripting language that is not imposed, possibly supplemented by APIs, frameworks and other add-ons. The code produced must not be dependent on the DBMS selected. The latter is not imposed. The application must be portable to an operating system different from the one it was produced on. In the context of portability to a mobile application environment, care must be taken to separate the user rendering from the application’s internal logic as much as possible.

  • Architecture: The architecture of the code produced must respect the conventions of use: development based on libraries of functions or MVC architecture.
